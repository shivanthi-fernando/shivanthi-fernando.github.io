My Design Process

My design process can vary depending on the project's goals, timeline, and constraints, the following steps represent the typical flow I follow when approaching a UX challenge:
Understand the Problem
+

I begin by understanding the problem, conducting user research, analyze competitors, create user personas. Also understand the business goals.

Define the Problem
+

From the details I gathered through the research, I clearly define the problem statement and create to be solved list to proceed

Ideate the Solution
+

I create user journeys to map out the overall experience. Then I create wireframes focusing on the information architecture.

Design and Prototype
+

I create high-fidelity designs and interactive prototypes to bring the user flow to life, ensuring the design is visually appealing, functional, user friendly and accessible.

Test and Validate
+

I test the prototypes with end users, get their feedback and do the necessary changes to make it more user friendly.
